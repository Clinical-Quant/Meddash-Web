| Thought Leader Timelines | @adamfeuerstein, @andybiotech, @BiotechCH | Every 6-12h | High |

| Bio/acc Community | Posts from community (until May 2026) | Daily | High |
| XChats Mentions | Monitor for migration | Daily | Medium |

**Step 2: Processing Pipeline**:
```
X API Read → Raw JSON Storage → Deduplication (Supabase) → LLM Scoring:
  - Relevance Score (0-10)
  - Clinical Insight Extraction
  - Pain Point Flagging
  - Lead Opportunity Score
→ Enrich with existing MedDash data (trials, KOLs, PDUFA)
→ Store in "X_Insights" table
```

**Step 3: LLM Scoring Prompt Template**:
```
You are analyzing a biotech/pharma tweet for KOL/therapeutic area intelligence value.

Tweet: {tweet_text}
Author: {author_handle}

Score (1-10) on:
1. KOL_RELEVANCE: Does this mention KOLs, key opinion leaders, medical affairs?
2. TA_RELEVANCE: Does this discuss therapeutic areas (oncology, immunology, etc.)?
3. CATALYST_SIGNAL: Does this indicate trial readouts, FDA dates, catalysts?
4. PAIN_POINT: Does this reveal a problem ("slow KOL mapping", "expensive tools")?
5. LEAD_POTENTIAL: Is the author likely to need MedDash briefs?

Output JSON:
{
  "scores": {"kol_relevance": X, "ta_relevance": X, "catalyst_signal": X, "pain_point": X, "lead_potential": X},
  "clinical_insight": "extracted insight or null",
  "pain_point_detected": "description or null",
  "recommended_action": "engage/monitor/ignore",
  "lead_score": 0-100
}
```
